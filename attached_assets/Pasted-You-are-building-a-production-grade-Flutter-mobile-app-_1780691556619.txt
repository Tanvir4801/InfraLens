═══════════════════════════════════════
SERVICES LAYER
═══════════════════════════════════════

api_service.dart — singleton with these methods:
  Future<MetricsSnapshot> fetchMetrics()
    → GET {baseUrl}/api/metrics
  Future<List<ServerInfo>> fetchServers()
    → GET {baseUrl}/api/servers
  Future<List<AlertModel>> fetchAlerts()
    → GET {baseUrl}/api/alerts
  Future<PredictionResult> fetchPrediction()
    → GET {baseUrl}/api/predict
  Future<List<Map>> fetchServerHistory(String name)
    → GET {baseUrl}/api/servers/{name}/history
  Future<IncidentReport> postIncidentReport(AlertModel alert)
    → POST {baseUrl}/api/incident-report
  Future<String> postAiChat(String question)
    → GET {baseUrl}/api/chat?q={question}
  Future<void> acknowledgeAlert(String id)
    → PATCH {baseUrl}/api/alerts/{id}/acknowledge
All methods: handle timeout (10s), catch SocketException,
return cached data from LocalStorageService on failure.

websocket_service.dart:
  Connects to {wsUrl}/ws/live
  Emits Stream<MetricsSnapshot>
  Auto-reconnects with exponential backoff:
    1s → 2s → 4s → 8s → 16s → 30s max
  Exposes connectionState: Stream<WsState>
    (WsState: connecting, connected, disconnected)

notification_service.dart:
  Init flutter_local_notifications on startup
  showCriticalAlert(AlertModel alert) — fires local
    notification with alert name + description
  compareAndNotify(List<AlertModel> newAlerts,
    List<AlertModel> oldAlerts) — diffs by alert ID,
    fires notification only for genuinely new criticals

local_storage_service.dart (shared_preferences):
  saveMetrics(MetricsSnapshot) / loadMetrics()
  saveAlerts(List<AlertModel>) / loadAlerts()
  savePinEnabled(bool) / loadPinEnabled()
  savePin(String) / loadPin()

═══════════════════════════════════════
PROVIDERS
═══════════════════════════════════════
MetricsProvider (ChangeNotifier):
  - Subscribes to WebSocketService stream
  - Maintains rollingHistory: List<double> (last 20 cpu values)
  - Tracks connectionState (WsState)
  - Computes healthScore (0-100):
      100 - (cpu*0.4 + ram*0.3 + disk*0.2 + alertPenalty*0.1)
      where alertPenalty = criticalCount * 15 + warningCount * 5

AlertsProvider (ChangeNotifier):
  - Fetches every 30s
  - Calls NotificationService.compareAndNotify on each fetch
  - Exposes filteredAlerts(String severity)
  - Tracks acknowledgedIds: Set<String>

ServersProvider / PredictionProvider: similar pattern.
